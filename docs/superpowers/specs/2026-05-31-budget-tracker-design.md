# Personal Budget Tracker — Design Spec

**Date:** 2026-05-31
**Status:** Approved

## Overview

A local Python pipeline that imports Korean spending data (국민카드, 네이버페이, 카카오페이) from CSV exports, categorizes transactions, and pushes structured data to Google Sheets. The Sheets workbook displays a monthly dashboard with expandable category rows.

---

## Architecture

```
[CSV 파일들]          [Python 파이프라인]           [Google Sheets]
국민카드.csv   ──→   파싱 → 정규화 → 분류 → 업로드 ──→  transactions
네이버페이.csv ──→   (per-source 파서)               merchant_categories
카카오페이.csv ──→                                   dashboard
```

### Execution Flow

1. User exports CSVs from 국민카드 app, 네이버페이, 카카오페이
2. Run: `python import.py --files kb.csv naver.csv kakao.csv`
3. Each parser reads its source CSV and returns a list of `Transaction` objects
4. Categorizer looks up each merchant in `merchant_categories` sheet
   - Found → use saved category
   - Not found → prompt user in CLI → save to sheet
5. Append transactions to `transactions` sheet (deduplication by date+amount+merchant)
6. Rebuild `dashboard` sheet with monthly summaries and row groups

## Google Sheets Structure

### Sheet 1 — `transactions`

Raw normalized transaction data. Append-only; never modified after import.

| Column | Type | Description |
|--------|------|-------------|
| date | YYYY-MM-DD | Transaction date |
| amount | integer | Amount in KRW (positive = expense) |
| merchant | string | Merchant name as-is from source |
| category | string | Assigned category |
| source | string | kb_card / naver_pay / kakao_pay |
| memo | string | Optional note from source file |

### Sheet 2 — `merchant_categories`

Persistent merchant → category mapping. Editable directly in Sheets.

| Column | Type | Description |
|--------|------|-------------|
| merchant | string | Exact merchant name |
| category | string | Category label (e.g., 카페, 식비, 쇼핑) |

### Sheet 3 — `dashboard`

Auto-generated by the Python script after each import. Rebuilt from scratch each run.

Layout (per month, descending):
```
▼ 2026년 5월
    카페          ₩45,000   ← category summary row
      스타벅스    ₩4,500    ← detail rows (row group, collapsible)
      이디야      ₩3,800
    식비          ₩120,000
      BBQ         ₩22,000
▶ 2026년 4월    ← collapsed by default
```

Uses Google Sheets **Row Groups** API for native +/- expand/collapse.

## Python Project Structure

```
personal-budget-tracker/
├── parsers/
│   ├── kb_card.py        # 국민카드 CSV parser
│   ├── naver_pay.py      # 네이버페이 CSV parser
│   └── kakao_pay.py      # 카카오페이 CSV parser
├── models.py             # Transaction dataclass
├── categorizer.py        # Merchant lookup + CLI prompt
├── sheets.py             # Google Sheets API wrapper
├── dashboard.py          # Dashboard sheet builder (row groups)
├── import.py             # Main entry point (argparse)
├── config.py             # SPREADSHEET_ID, credentials path
└── requirements.txt
```

### Component Responsibilities

**`models.py`**
- `Transaction` dataclass: `date, amount, merchant, category, source, memo`

**`parsers/`**
- One file per source; each exports a single function `parse(filepath) -> list[Transaction]`
- Handles source-specific column names, date formats, encoding (EUC-KR vs UTF-8)
- Sets `category = ""` (empty); categorizer fills this in

**`categorizer.py`**
- Reads `merchant_categories` sheet into a dict on startup
- `categorize(transactions, sheets_client) -> list[Transaction]`
- For unknown merchants: `inquirer`-style CLI prompt with free-text input
- Batches new mappings and writes to sheet at end of run

**`sheets.py`**
- Handles OAuth2 service account authentication
- Provides: `read_sheet()`, `append_rows()`, `clear_and_write()`, `apply_row_groups()`
- Single responsibility: Sheets API calls only; no business logic

**`dashboard.py`**
- Reads all transactions from `transactions` sheet
- Groups by year-month → category → individual transactions
- Writes dashboard rows to Sheet 3
- Applies Row Groups via `batchUpdate` (sheets API `addDimensionGroup`)

**`import.py`**
- `argparse`: `--files` (required, one or more CSV paths)
- Orchestrates: parse → categorize → append transactions → rebuild dashboard
- Deduplicates before appending: skip rows where (date, amount, merchant) already exists

## Authentication

- Google OAuth2 **service account** JSON key file
- Path configured in `config.py` (e.g., `credentials.json` in project root)
- Service account email must be given Editor access to the Spreadsheet
